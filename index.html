Some Recent Work

Development

Tag Checker

"Tag Checker" is a headless application built using AWS services (primarily a Step Function state machine) in order to iterate through all AWS services created in an account and verify that the tagging methodology of developers spinning up resources are meeting tagging standards. If tagging standards aren't met, the resource is deleted and a record is logged in an RDS db for further functionality. Click on the link to see the iterator state functional code and the state machine definition.

Branching Strategy

I recently designed and implemented a new branching strategy for development teams. This branching strategy was designed in order to account for hotfix (bug) releases, while allowing dev teams to continue with production work with less concern for code overwrites. Additionally, instituted an AWS(SES)-based email notification service when hot fix branches are created in CodeCommit with the purpose of instructing developers to integrate hotfix code with new production code.
